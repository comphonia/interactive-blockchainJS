An Overview of Blockchain Technology

What is Blockchain?

Blockchain technology is the new buzzword in the business world. It is a simply list of transactions called blocks, each block contains a reference to the previous block, a timestamp and its transaction data.

How does Blockchain work?

A Blockchain network is comprised of nodes, they all listen for transactions in the network, each node has a copy of the ledger (a register of chronological events that happened on the network), these events are broken down into blocks that contain a group of transactions. These blocks are linked in a way that one references the next.

To further break this down for the layman;
Equate Blockchain technology to a ledger or a sign-in sheet, every person that signs in has to write their name, time and any other relevant information at the time of sign-in, each entry would be considered data or a block, every entry to the ledger does not overwrite the previous ones.

Attributes of a Blockchain

A properly implemented blockchain system needs to have some attributes before it is deployed, it must be secure, valid, true and relevant. Other features can be implemented such as immutability, verification and tracking to further strengthen the system.

Security

All features below means nothing if the system is not secure, this is a priority. Secure databases, utilize encryption, integrate authorization & authentication, among others.

Validation

Check if the user has the money to spend or data to share
Prevent a double spend (utilizing the same currency before the initial transaction is completed)
Disregard invalid transactions

Truthfulness

Ensure truthfulness of a transaction with cryptographic signatures

Relevancy

Prevent Distributed Denial of Service (DDOS) attacks on the network by implementing a mechanism for charging per transaction. A fee in addition to proof of work and proof of stake can reduce spamming.

Immutability

Ensure that blocks added to the chain can not be changed.

Verification

Add some sort of authentication to verify both the sender and receiver and be sure to store both their e-addresses.

Proof of work

Proof is an economic measure that prevents DDOS attacks and service abuses by requiring value from the miner. The work will be solving a cryptographic function (CPU cost function). The miner will have to utilize massive CPU/GPU cycles to find a nonce (repeatedly hashing a block and checking if the resulting value has a required number of zeros, determined by the difficulty level.)
Proof of work mostly exists in public implementations of Blockchain, The proof is easy for verification but hard to generate so incentives are built into the system to reward miners who submit new blocks. The reward is the currency being mined, this currency also increases the supply to the network
